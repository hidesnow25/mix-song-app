# 音声合成アプリ

## 2つの音声ファイルを次の選択肢から選んで再生できるファイルに合成する

- 左側からのみ再生
- 右側からのみ再生
- 左右から再生



### 将来的に左右から再生できる音量の調整ができるように３つの選択しではなくスライダーで調整できるUIを検討する

## 下記の機能も検討する
- ファイルをドラッグ＆ドロップに対応する
- ソースはGithubに入れる
- デプロイの自動化機能を付ける
- サーバ側の料金をかけたくないのでフロントエンドだけで機能する(サーバ側に移植できるように実装は移植にも対応できるようにしておく)
- 音声のプレビューのUIを付ける
- 音声を合成してダウンロードできる機能を付ける


## 開発の言語はライブラリを基準に選定する
- フロントエンドで音声ファイルを左右から分けて再生できるように操作しやすいUIを作りやすいライブラリがあるプログラミング言語を選ぶ
- typescript,node.js,pythonから優先的に選択する
- 選択した音声ファイルの波形データを表示してドラッグで範囲を選択して無音にすることで音声ファイルをパートに分けて合成できるUIを検討する
- 例えば１つの曲をソロで歌唱する音声ファイルが２つ存在した場合各歌唱パートで分けて歌唱させたり２人でハモリのようなパートを作ったり出来るようにする


### 修正内容
- 音声ファイルAを左側からのみ再生
- 音声ファイルBを右側からのみ再生
- 音声ファイルAとBを合成して左側からは音声ファイルAが右側からは音声ファイルBが再生される音声ファイルCを合成する
- ダウンロードする音声ファイルの拡張子を選択できるようにする
- デフォルトはインプットに使われた音声ファイルの拡張子と同じにする
- 音声の波形は左右にならべるのではなく上下に並べるようにする


# 追加の修正内容
- 一度音声ファイルの波形は非表示にする(後の開発で利用するため機能は残す)
- フロントで処理中の場合はローディングを入れる（画面全体ではなく分かりやすい見た目で入れる）
- 左側からのみ再生と右側からのみ再生を消す
- 音声ファイルAを左側からのみ音声ファイルBを右側からのみ再生できるようにするか音声ファイルAとBを左右どちらともから再生できるようにするかのボタンのみにする
- 左右から再生で生成した音声ファイルをダウンロードして確認していると音声ファイルBを無音にしたところで右側から音声ファイルAの音声が音量が小さいが聴こえたのはなぜ？完全に左右別々から音声を再生することはできない？

# 速度改善・ファイル名機能
- MP3書き出しを純JSエンコーダー（lamejs）からWebAssembly版LAME（wasm-media-encoders）に変更
- エンコード処理をWeb Workerに移し、メインスレッドを一切ブロックしないようにした
- 実測で45秒の音声のエンコード時間が約2.7〜3.0秒→約0.7〜1.1秒に短縮（約3〜4倍高速化）、エンコード中も画面は約60fpsで応答し続けることを確認
- ダウンロードするファイル名を入力できるフォームを追加
- デフォルトのファイル名は音声ファイルAとBのファイル名（拡張子を除く）をハイフンでつないだもの
- ファイル名にOSで使えない文字（`/ \ : * ? " < > |`など）が含まれる場合は、ダウンロード時のみ自動的に置換される（入力欄自体は編集の邪魔にならないようそのまま表示）
